**프로젝트명**: public-rental-housing

**최종 수정일**: 2026-04-05
**상태**: 피드백 반영 (v1.1)

## 1. 프로젝트 배경 및 목적

### 1.1 문제 정의

공공임대주택(SH, LH, 마이홈 등) 공고는 여러 기관 사이트에 분산되어 있고, 공고문이 PDF로 제공되며, 자격 조건이 복잡하여 일반인이 자신에게 해당하는 공고를 빠르게 파악하기 어렵다.

현재 사용자가 겪는 불편:
- 여러 사이트를 일일이 방문하여 새 공고를 확인해야 함
- 공고문 PDF를 열어 복잡한 자격 조건을 직접 해석해야 함
- 본인의 소득, 자산, 가구 구성 등에 맞는 공고인지 스스로 판단해야 함
- 공고 마감일을 놓치기 쉬움

### 1.2 목표

1. **🔴 마감일을 놓치지 않는 것이 최우선**: D-Day 카운트다운, 마감 임박 강조, 리마인더 알림으로 접수 기한을 절대 놓치지 않도록 지원
2. **공고 자동 수집**: SH, LH, 마이홈 등 공공임대 공고를 자동으로 수집하여 한 곳에서 확인
3. **자격 조건 분석**: 공고문의 자격 조건을 패턴 기반 파서(정규식+키워드)로 추출하여 사용자 프로필과 자동 매칭
4. **입지 분석**: 근무지 기준 통근 시간, 희망 면적 등 주거 선호 조건까지 고려한 맞춤 필터링
5. **맞춤 알림**: 새 공고 + 마감 임박 리마인더를 이메일로 알림
6. **무료 운영**: 서비스 운영 비용을 0원에 가깝게 유지
7. **파이프라인 신뢰성**: 크롤링/분석/알림 각 단계의 정상 동작을 모니터링하고 검증

## 2. 사용자 정의

### 2.1 사용자 유형

| 유형 | 설명 | 인증 필요 |
|------|------|-----------|
| **비회원 방문자** | 공고 목록만 열람하는 사용자 | 불필요 |
| **회원 (로그인 사용자)** | 프로필 등록 후 자격 필터링, 알림을 사용하는 사용자 | 필요 |

### 2.2 사용자 프로필 데이터 모델

회원이 입력하는 자격 판단용 정보:

> **⚠️ 소득 기준 안내**: 모든 소득 항목은 **세전(세금 공제 전) 월평균 소득** 기준입니다.
> 공공임대주택 자격 판정에 사용되는 "도시근로자 가구당 월평균 소득"이 세전 기준이므로 이에 맞춥니다.

| 필드 | 타입 | 설명 | 필수 여부 |
|------|------|------|-----------|
| 이름 | string | 사용자 이름 | 필수 |
| 생년월일 | date | 나이 계산용 | 필수 |
| 가구 유형 | enum[] | **복수 선택 가능**. 1인 가구, 예비 신혼부부, 신혼부부, 기혼(자녀 있음), 다자녀, 한부모, 노부모 부양 등. 예: 미혼이지만 혼인 예정인 경우 "1인 가구 + 예비 신혼부부" 동시 선택 → 두 유형 모두에 해당하는 공고를 필터링 | 필수 |
| 혼인 여부 | enum | 미혼, 혼인 예정, 기혼, 이혼 | 필수 |
| 혼인 예정일 | date \| "미정" | 예비 신혼부부의 경우. **"미정" 선택 가능** — 당첨 후 입주 전 혼인신고 예정인 경우 해당 | 선택 |
| 가구원 수 | number | 주민등록등본 기준, 본인 포함. **ⓘ 도움말 버튼 제공** (아래 2.4 참조) | 필수 |
| 세대주 여부 | boolean | 주민등록등본상 세대주인지 여부 | 필수 |
| 무주택 기간 | number (개월) | 무주택 세대 구성원으로서의 기간 | 필수 |
| 본인 월평균 소득 | number (만원) | **세전(세금 공제 전)** 본인의 월평균 근로소득. 근로소득원천징수영수증 또는 건강보험료 산정 기준 참고 | 필수 |
| 총 자산 | number (만원) | 부동산 + 금융(예적금, 주식 등) + 기타 자산 합계 | 필수 |
| 자동차 가액 | number (만원) | 보유 차량 보험개발원 기준 가액. 미보유 시 0 입력 | 필수 |
| 청약통장 유형 | enum | 청약저축, 주택청약종합저축, 미가입 | 필수 |
| 청약통장 가입일 | date | 가입 기간 자동 계산용 | 조건부 |
| 청약통장 납입 횟수 | number | 매월 납입 인정 횟수 | 조건부 |
| 현재 주소 | string | 시/도/시군구/읍면동 단위 (주민등록등본 기준 실제 주소) | 필수 |
| 관심 지역 | string[] | 시/도 또는 시군구 단위 복수 선택 | 필수 |
| 이메일 | string | 알림 수신용 | 필수 |

### 2.3 배우자(예정자) 정보

> 가구 유형에 "예비 신혼부부", "신혼부부", "기혼" 중 하나 이상 포함된 경우 표시

| 필드 | 타입 | 설명 | 필수 여부 |
|------|------|------|-----------|
| 배우자 생년월일 | date | 나이 계산용 | 필수 |
| 배우자 월평균 소득 | number (만원) | **세전(세금 공제 전)** 배우자의 월평균 근로소득 | 필수 |
| 배우자 총 자산 | number (만원) | 부동산 + 금융 + 기타 자산 합계 | 필수 |
| 배우자 근무지 | string | 시/도/시군구 또는 역명 (통근 분석용) | 선택 |

### 2.4 가구원 수 도움말 (UI 툴팁/모달로 제공)

> **가구원 수란?**
> 주민등록등본에 함께 등재된 사람의 수입니다 (본인 포함).
>
> **포함 대상**: 본인, 배우자, 직계존속(부모/조부모), 직계비속(자녀)
> **미포함**: 형제자매, 친구, 동거인 등 (등본에 있어도 직계가 아니면 제외되는 공고 있음)
>
> **예시**:
> - 본인 혼자 세대 구성 → **1명**
> - 본인 + 부모 동거 → **3명** (본인 1 + 부모 2)
> - 본인 + 배우자 + 자녀 1명 → **3명**
>
> **예비 신혼부부의 경우**: 공고에 따라 혼인 후 합가 기준(본인 + 배우자 = 2명)으로 산정하기도 합니다. 공고별 자격 분석에서 이를 구분하여 안내합니다.
>
> ※ 정확한 산정 기준은 공고마다 다를 수 있으므로, 원본 공고문을 반드시 확인하세요.

### 2.5 주거 선호 조건 (맞춤 필터링 및 입지 분석용)

| 필드 | 타입 | 설명 | 필수 여부 |
|------|------|------|-----------|
| 희망 전용면적 | range (㎡) | 최소~최대 전용면적. **ⓘ 평수 환산 도움말**: 10평≈33㎡, 15평≈49.5㎡, 20평≈66㎡ | 선택 |
| 본인 근무지 | string | 시/도/시군구 또는 역명 입력. 예: "마천역(5호선)" | 선택 |
| 배우자(예정) 근무지 | string | 2.3과 동일 필드 (배우자 정보에서 입력) | 선택 |
| 희망 최대 통근 시간 | number (분) | 대중교통 기준 편도 최대 시간. 예: 60분 | 선택 |
| 최대 보증금 | number (만원) | 감당 가능한 최대 보증금 | 선택 |
| 최대 월임대료 | number (만원) | 감당 가능한 최대 월임대료 | 선택 |

### 2.6 참조 사용자 시나리오 (페르소나)

> 개발 및 테스트 시 기준이 되는 실제 사용자 시나리오.

**사용자 A (본인)**:
- 가구 유형: **1인 가구 + 예비 신혼부부** (복수 선택)
- 혼인 여부: 혼인 예정 / 혼인 예정일: **미정** (당첨 후 입주 전 혼인신고 계획)
- 근무지: **마천역 (5호선)**
- 관심 권역: 서울 동남부 (위례, 거여, 마천, 복정, 하남, 성남 구도심 등)

**사용자 A의 배우자(예정)**:
- 현재 근무지: **수원**
- 1~2년 내 **서울 동남권 인접 경기(성남, 하남 등)** 초등학교로 전근 예정
- 입지 분석 시: 본인 마천역 + 배우자 성남/하남 기준 직주근접 분석

**입지 분석 핵심 요구**:
- 본인 마천역 기준 대중교통 통근 시간
- 배우자 현재 수원 및 예상 전근지(성남/하남) 기준 통근 시간
- 최적 입지: 위례, 거여, 마천, 복정, 하남, 성남 구도심 등 서울 동남부 권역 집중

## 3. 기능 요구사항

### 3.1 공고 수집 (Scraping Pipeline)

#### FR-001: 공고 목록 크롤링
- SH공사, LH공사, 마이홈 포털의 공공임대 공고 목록 페이지를 크롤링
- 수집 항목: 공고명, 공고일, 접수 기간, 주택 유형, 지역, 공고 상세 URL, 공고 기관
- GitHub Actions cron으로 **4시간마다** 실행 (하루 6회, 월 약 180회)
- GitHub Actions 무료 한도: 월 2,000분 → 1회 실행 약 3~5분 가정 시 월 900분, 한도 내 충분

#### FR-002: 공고 상세 크롤링
- 공고 목록에서 새로 감지된 공고의 상세 페이지를 크롤링
- 수집 항목: 공급 세대수, 전용면적, 임대조건(보증금/월임대료), 자격 요건 텍스트, 일정(접수일, 서류제출일, 당첨발표일)

#### FR-003: PDF 공고문 처리
- 공고에 첨부된 PDF 공고문 다운로드
- PDF에서 텍스트 추출
- 추출된 텍스트를 DB에 저장 (자격 분석 입력용)

#### FR-004: 중복 감지
- 이미 수집된 공고는 재수집하지 않음
- 공고의 고유 식별자(공고번호 또는 URL)로 중복 판단

#### FR-005: 공고 변경 감지
- 기존 공고 내용이 변경(수정공고)된 경우 감지
- 변경 이력 저장

---

### 3.2 자격 분석 엔진 (패턴 기반)

#### FR-006: 자격 조건 추출
- 공고문 텍스트에서 자격 조건 블록을 추출
- 정규식 + 키워드 매칭 기반 패턴 파서로 구조화된 JSON으로 변환 (외부 API 과금 없음)
- 추출 대상 조건:
  - 소득 기준 (도시근로자 월평균 소득 비율)
  - 자산 기준 (총 자산, 자동차 가액)
  - 가구 유형 요건 (1인 가구, 신혼부부, 다자녀 등)
  - 무주택 기간
  - 연령 제한
  - 거주 지역 요건
  - 청약통장 요건 (가입 기간, 납입 횟수)
  - 특별 조건 (입주 전 혼인신고, 소득 활동 기간 등)

#### FR-007: 자격 매칭
- 추출된 자격 조건과 사용자 프로필을 비교
- 판정 결과:
  - **적격**: 모든 조건 충족
  - **확인 필요**: 대부분 충족하나 일부 조건 확인 필요 (예: "입주 전 혼인신고 시 가능")
  - **부적격**: 핵심 조건 미충족
- 각 조건별 충족/미충족 상세 결과 제공

#### FR-008: 분석 효율 관리
- 동일 공고에 대한 중복 분석 방지
- 분석 결과 캐싱 (공고 조건은 변경되지 않는 한 재분석 불필요)
- 패턴 기반 파서이므로 외부 API 비용 $0

### 3.3 웹 대시보드

#### FR-009: 메인 대시보드 (비로그인)
- 최신 공고 목록 표시
- 공고 기본 정보: 공고명, 기관, 지역, 접수 기간, 주택 유형
- 공고 상태 뱃지: 접수중, 접수예정, 마감
- **⏰ D-Day 표시**: 접수 마감까지 남은 일수를 눈에 띄게 표시 (예: "D-3", "D-Day", "마감")
- **🔴 마감 임박 강조**: 마감 3일 이내 공고는 빨간색 강조 표시, 상단 고정
- 정렬 기본값: **마감임박순** (마감이 가까운 공고가 먼저 표시)
- 추가 정렬: 최신순, 지역순
- 필터: 지역, 주택 유형, 기관, 상태, 전용면적 범위
- 검색: 공고명 키워드 검색

#### FR-010: 공고 상세 페이지 (비로그인)
- 공고 기본 정보 전체 표시
- **⏰ D-Day 카운트다운** 상단 고정 표시 (접수 마감일까지)
- 공급 세대수, 전용면적 (㎡ + 평수 환산 병기), 임대 조건
- 주요 자격 조건 요약 (패턴 분석 결과 기반)
- **주요 일정 타임라인**: 공고일 → 접수시작 → **접수마감(강조)** → 서류제출 → 당첨발표 → 계약
- 원문 링크 (해당 기관 사이트로 이동)

#### FR-010-1: 입지/통근 분석 (로그인 시 추가 표시)
- 사용자 근무지로부터 해당 주택까지 **대중교통 예상 통근 시간** 표시
- 배우자(예정자) 근무지 통근 시간도 함께 표시
- 통근 시간이 사용자 희망 시간 이내인지 적합/부적합 뱃지
- 지도에 주택 위치, 근무지, 통근 경로 시각화 (Phase 4에서 구현)

#### FR-011: 내 자격 필터링 (로그인 필수)
- 대시보드에서 "내 조건에 맞는 공고만 보기" 토글
- 공고 목록에 적격/확인필요/부적격 뱃지 표시
- 적격 공고 상단 정렬
- 공고 상세에서 "내 자격 분석" 탭: 각 조건별 충족/미충족 상세

#### FR-012: 사용자 프로필 관리 (로그인 필수)
- 프로필 입력 폼 (2.2에 정의된 필드)
- 배우자 정보 입력 (가구 유형이 신혼부부/예비 신혼일 때 표시)
- 관심 지역 복수 선택
- 프로필 수정
- 프로필 변경 시 기존 공고에 대해 자격 재분석

#### FR-013: 알림 설정 (로그인 필수)
- 이메일 알림 ON/OFF 토글
- 알림 조건 설정: 적격 공고만 / 적격 + 확인필요
- 알림 빈도: 새 공고 발생 시 즉시 / 하루 1회 요약

---

### 3.4 알림 시스템

#### FR-014: 이메일 알림 — 새 공고
- Gmail SMTP를 사용한 알림 메일 발송
- 새 공고 감지 시, 해당 공고가 사용자의 자격 조건에 맞으면 이메일 발송
- 이메일 내용: 공고명, 기관, 접수 기간, **D-Day**, 적격 판정 요약, 대시보드 링크
- 발송 제한: Gmail SMTP 하루 500건 무료

#### FR-014-1: 이메일 알림 — 마감 임박 ⭐ (핵심 기능)
- 접수 마감 **3일 전**, **1일 전**, **당일** 리마인더 이메일 자동 발송
- 사용자가 적격/확인필요로 판정된 공고 중 아직 접수하지 않은 공고 대상
- 이메일 제목에 긴급도 표시: "[D-3] 행복주택 마곡 접수 마감 임박", "[D-Day] 오늘 마감!"
- 사용자가 리마인더 시점을 커스텀 설정 가능 (기본: D-3, D-1, D-Day)

#### FR-015: 알림 히스토리
- 발송된 알림 내역 조회 (알림 센터)
- 읽음/안읽음 상태 관리

### 3.5 파이프라인 모니터링 (Health Check)

#### FR-018: 크롤링 상태 모니터링
- 각 크롤링 실행의 성공/실패 로그 기록
- 연속 실패 시 관리자(본인)에게 이메일 알림
- 모니터링 대시보드 (관리자용): 최근 크롤링 실행 이력, 수집 건수, 에러 로그
- GitHub Actions 실행 로그 링크 제공

#### FR-019: 분석 품질 검증
- 파서가 추출한 자격 조건을 사람이 검증할 수 있는 UI 제공 (관리자용)
- 원본 공고문 텍스트와 파서 분석 결과를 나란히 비교
- 검증 상태: 미검증 / 정확 / 수정필요
- 수정이 필요한 경우 수동 보정 후 저장
- 분석 정확도 통계: 전체 공고 중 정확/수정필요 비율 추적

#### FR-020: 파이프라인 전체 상태 요약
- 대시보드에 파이프라인 상태 표시 (관리자용):
  - 마지막 크롤링 실행 시각
  - 마지막 자격 분석 실행 시각
  - 최근 24시간 수집 공고 수
  - 미분석 공고 수 (크롤링됐지만 아직 자격 분석 안 된 것)
  - 이메일 발송 성공/실패 건수

## 4. 비기능 요구사항

### 4.1 성능
- 대시보드 메인 페이지 로딩: 3초 이내
- 공고 목록 필터링/정렬: 1초 이내 (클라이언트 사이드)

- HTTPS 필수 (Vercel 기본 제공)

### 4.3 가용성

### 4.4 비용 제약
- 월 운영비 목표: **0원** (모든 서비스 무료 티어 범위 내, 외부 API 과금 없음)
- 패턴 기반 파서 사용으로 분석 비용 $0

## 9. 에이전틱 파이프라인 상세

### 9.1 크롤링 → 분석 → 알림 → 모니터링 흐름

```
Step 1: 크롤링 에이전트
  - Playwright로 SH/LH/마이홈 공고 목록 페이지 접근
  - 신규 공고 감지 (DB의 기존 external_id와 비교)
  - 신규 공고 상세 페이지 크롤링
  - PDF 다운로드 및 텍스트 추출
  - 결과를 announcements 테이블에 저장
  - 실행 로그를 pipeline_logs에 기록

Step 2: 분석 엔진 (패턴 기반)
  - 새로 적재된 공고의 pdf_text를 입력으로 받음
  - 패턴 기반 파서로 자격 조건 추출:
    * 정규식으로 소득/자산/연령/무주택 기간 등 수치 조건 추출
    * 키워드 매칭으로 가구 유형, 지역 요건, 청약통장 조건 등 분류
    * 섹션 분할로 대상 그룹별 조건을 구분하여 구조화
    * Output: 구조화된 자격 조건 JSON
  - 결과를 eligibility_conditions 테이블에 저장
  - 실행 로그를 pipeline_logs에 기록

Step 3: 매칭 에이전트
  - 모든 회원의 프로필을 가져옴
  - 새 공고의 자격 조건과 각 사용자 프로필을 비교
  - 가구 유형 복수 선택 반영: 사용자의 모든 가구 유형에 대해 각각 매칭
  - 적격/확인필요/부적격 판정
  - 주거 선호 조건 추가 매칭: 면적, 통근시간, 보증금/월임대료
  - 결과를 eligibility_results 테이블에 저장

Step 4: 알림 에이전트 — 새 공고 알림
  - 매칭 결과 중 적격/확인필요인 사용자 필터링
  - 해당 사용자의 알림 설정 확인 (enabled, level, frequency)
  - 조건에 맞으면 이메일 발송
  - 발송 결과를 notifications 테이블에 저장

Step 5: 알림 에이전트 — 마감 리마인더 ⭐
  - 매일 1회 실행 (별도 cron)
  - 접수 중인 공고 중 마감일이 사용자의 리마인더 설정(D-3, D-1, D-Day)에 해당하는 것 필터링
  - 해당 공고가 사용자에게 적격/확인필요인 경우 리마인더 이메일 발송
  - "이 공고 접수 마감이 3일 남았습니다" 형태의 긴급 알림

Step 6: 모니터링 에이전트 (Health Check)
  - 각 Step의 pipeline_logs를 확인
  - 연속 2회 이상 실패한 파이프라인이 있으면 관리자에게 이메일 알림
  - 미분석 공고(크롤링됐지만 자격 분석 안 된 것)가 있으면 재분석 트리거
  - 일일 요약 리포트 생성 (수집 건수, 분석 건수, 알림 발송 건수, 에러 건수)
```

### 9.2 패턴 기반 파서 설계 (요약)

```
[파서 파이프라인]
1. 텍스트 전처리: PDF 추출 텍스트 정규화 (공백/줄바꿈 정리, 특수문자 처리)
2. 섹션 분할: "신청자격", "입주자격", "소득기준" 등 키워드로 자격 조건 블록 식별
3. 대상 그룹 분류: "1인 가구", "신혼부부", "다자녀" 등 가구 유형별 섹션 구분
4. 조건 추출 (정규식 + 키워드 매칭):
   - 소득: "월평균소득의 (\d+)%" 패턴
   - 자산: "총자산 (\d+,?\d*)만원" 패턴
   - 자동차: "자동차가액 (\d+,?\d*)만원" 패턴
   - 연령: "만 (\d+)세 이상", "(\d+)세 이하" 패턴
   - 무주택: "무주택 세대 구성원", "(\d+)년 이상 무주택" 패턴
   - 지역: "서울특별시 거주", "해당 지역 거주" 등 키워드
   - 청약통장: "가입 후 (\d+)개월", "(\d+)회 이상 납입" 패턴
5. 구조화 출력: JSON 형태로 변환

[Output Schema]
target_groups: array of {
    group_name: string,
    income_limit_percent: number,
    asset_limit: number,
    car_limit: number,
    age_range: { min: number, max: number },
    homeless_months_min: number,
    region_requirements: string[],
    subscription_months_min: number,
    household_types: string[],
    marriage_conditions: string,
    special_conditions: string[]
  }
```

## 12. 면책 및 법적 고지

- 본 서비스는 공공임대주택 공고 정보를 편리하게 제공하기 위한 **비공식** 서비스임
- 자동 자격 분석 결과는 **참고용**이며, 정확한 자격 여부는 해당 기관에 직접 확인 필요
- 공고 원문은 각 기관의 공식 사이트를 기준으로 함
- 데이터 수집 시 각 사이트의 robots.txt 및 이용약관을 준수함
- 사용자 개인정보는 자격 분석 목적으로만 사용되며, 제3자에게 제공하지 않음
