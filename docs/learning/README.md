## 🔑 핵심 개념 요약

### Git & GitHub
```
Git = 코드의 "시간 여행기계"
GitHub = Git을 저장하고 협업하는 웹서비스
GitHub Actions = 코드 변경 시 자동으로 일을 해주는 봇
```

### Supabase
```
Supabase = 관리형 PostgreSQL (귀찮은 것 다 해줌)
스키마 = 데이터베이스의 구조 (테이블 정의)
마이그레이션 = 스키마 변경을 코드로 기록하는 것
```

### Vercel
```
Vercel = 당신의 Next.js 앱을 인터넷에 공개해주는 호스팅 서비스
배포 = 로컬 코드를 인터넷에 올리는 것
빌드 = 배포하기 전에 코드를 최적화하는 과정
```

## 💡 자주 하는 질문

### Q: 왜 이 3가지 (Git, Vercel, Supabase)를 썼어?
**A**: 각각 무료, 쉬움, 자동화 가능하고, 서로 잘 어울리기 때문

### Q: 로컬에서 개발할 때 Vercel/Supabase가 필요한가?
**A**: 아니다. 로컬에서는 .env.local만 있으면 충분. 배포할 때만 필요

### Q: 환경변수는 왜 노출 금지인가?
**A**: 비밀번호 같은 것. 노출되면 누구든 데이터베이스 접근 가능

### Q: 마이그레이션이 실패하면?
**A**: Vercel 배포 로그에서 에러 메시지 확인. 대부분 데이터베이스 연결 문제

### Q: 배포 후 사이트가 안 보이면?
**A**: 
1. Vercel 배포 상태 확인
2. 빌드 로그에서 에러 찾기
3. 환경변수 설정 확인
