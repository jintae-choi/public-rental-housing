# 📚 공공임대주택 대시보드 - 기술 아키텍처 개요

## 당신이 직접 설정한 것 vs 제가 자동화한 것

### 🔵 당신이 직접 설정한 것 (수동 작업)
1. **Supabase 프로젝트 생성** - 웹 대시보드에서
2. **Supabase Connection String 복사** - 데이터베이스 연결 정보
3. **GitHub 토큰 업데이트** - workflow scope 추가
4. **GitHub Actions Secrets 추가** - 3개의 보안 정보 등록
5. **Vercel 대시보드에서 배포 연결** - 최종 배포 설정

### 🟢 제가 자동화한 것 (코드로)
- Git 초기화 및 커밋 관리
- GitHub Actions 워크플로우 파일 생성
- Vercel 배포 설정 파일
- Drizzle 스키마 생성
- 마이그레이션 파일 생성

## 전체 기술 스택의 역할

```
┌─────────────────────────────────────────────────────────────────┐
│                        당신의 개발 환경 (로컬)                      │
│  VS Code → pnpm install → pnpm dev → http://localhost:3000     │
└─────────────────┬───────────────────────────────────────────────┘
                  │ git push
                  ↓
┌─────────────────────────────────────────────────────────────────┐
│                     GitHub (코드 저장소)                          │
│  - 코드 버전 관리                                                 │
│  - GitHub Actions (자동화 워크플로우)                             │
│    └─ 매일 크롤링 실행                                            │
│    └─ DB 마이그레이션 자동 실행                                   │
└─────────────────┬───────────────────────────────────────────────┘
                  │ webhook
                  ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Vercel (배포 서버)                             │
│  - Next.js 앱 배포                                               │
│  - 환경변수 관리                                                  │
│  - SSL/도메인 자동 관리                                           │
│  - 배포 후 DB 마이그레이션 자동 실행                              │
└─────────────────┬───────────────────────────────────────────────┘
                  │
                  ↓
┌─────────────────────────────────────────────────────────────────┐
│                  Supabase (데이터베이스)                          │
│  - PostgreSQL 데이터베이스                                        │
│  - 실시간 데이터 동기화                                           │
│  - 사용자 인증 관리                                               │
│  - 자동 백업                                                      │
└─────────────────────────────────────────────────────────────────┘
```

---

## 왜 이 스택을 선택했는가?

| 기술 | 역할 | 왜 선택? |
|------|------|---------|
| **Git/GitHub** | 코드 버전 관리 + 자동화 | 협업, 이력 관리, CI/CD의 중심 |
| **GitHub Actions** | 자동 배포 + 크롤링 스케줄 | 무료, 코드와 같은 곳에서 관리 |
| **Vercel** | 프로덕션 배포 | Next.js 최적화, 자동 마이그레이션, 무료 호스팅 |
| **Supabase** | 데이터베이스 + 인증 | PostgreSQL + 관리형 서비스, 실시간 기능 |
| **Drizzle ORM** | 타입안전 쿼리 | TypeScript 100% 지원, 마이그레이션 관리 |

## 핵심 개념 정리

### 🔹 배포 (Deployment)
- **개념**: 로컬 개발 환경의 코드를 인터넷에서 접근 가능하게 공개
- **왜?**: 사용자들이 앱을 사용하려면 인터넷에 연결된 서버에 앱이 떠있어야 함
- **예**: Vercel이 당신의 코드를 받아 서버에 올리는 것

### 🔹 환경변수 (Environment Variables)
- **개념**: 코드에 노출되면 안 되는 민감한 정보 (DB 비밀번호, API 키 등)
- **왜?**: 보안, 로컬/테스트/프로덕션 환경마다 다른 값 사용
- **예**: `.env.local`은 로컬만, Vercel의 환경변수는 프로덕션 서버

### 🔹 자동화 (Automation)
- **개념**: 반복되는 작업을 자동으로 실행되도록 설정
- **왜?**: 시간 절약, 인적 오류 방지, 항상 일관된 결과
- **예**: `git push` → 자동으로 배포 + DB 마이그레이션 실행
